# Issue 004: 大模型润色过度改变原意，需支持自定义润色程度与提示词

## 背景
当前 `ExternalAPIProcessor.generateDiary()` 中使用固定的系统提示词和生成要求，导致大模型在实际执行中：
- 过度添加过渡语句和"温暖修饰"
- 改写力度过大，丢失了用户原始记录中的具体措辞、语气和细节
- 生成的日记像"AI 写作文"，而非用户真实表达的整理

用户核心诉求：**保留原意的前提下做最小必要整理**，并且用户希望自己能控制润色程度。

## 现状分析
当前固定提示词（`ExternalAPIProcessor.kt`）：
```
请基于以下碎片记录，生成一篇连贯、温暖、真诚的日记。
要求：
1. 保留原始记录的情感色彩和关键信息
2. 去除重复和冗余，但不过度改写
3. 使用第一人称，语气自然亲切
4. 适当添加过渡语句使文章流畅
5. 总字数控制在 300-2000 字之间
6. 不要编造碎片中没有的信息
```

问题点：
- **"适当添加过渡语句"** 和 **"温暖、真诚"** 的语义边界模糊，模型容易过度发挥
- 没有分级控制：用户无法选择"仅排版"、"轻度润色"、"深度改写"等不同档位
- 无法查看/修改底层发送给模型的完整提示词

## 期望功能

### 1. 润色程度分级（预设档位）
在设置页或生成日记前提供选项：

| 档位 | 说明 | 对提示词的影响 |
|---|---|---|
| **原文拼贴** | 几乎不改，仅按时间顺序拼接并分段 | 禁止改写、禁止添加过渡句、禁止总结 |
| **轻度整理** | 去除明显的重复和口癖，保留原句 | 允许删减冗余，禁止扩写、禁止改变措辞 |
| **自然润色** | 在保留原意基础上让语句更通顺 | 当前默认档位，可小幅调整措辞、添加必要过渡 |
| **深度改写** | 允许模型更大自由度地重组和发挥 | 当前实际效果接近此档 |

### 2. 自定义提示词（高级模式）
- 在设置页提供"自定义生成提示词"文本框
- 预填充当前默认提示词，允许用户自行修改
- 提示词模板支持变量占位，如 `{{fragments}}`、`{{date}}`、`{{style}}`
- 提供"恢复默认"按钮

### 3. 生成前预览（可选增强）
- 在正式生成前，先展示模型对碎片的"理解摘要"（关键词、时间线、情绪标签）
- 用户确认后再生成正文，降低"生成出来完全不是我想表达的"的挫败感

### 4. 提示词版本管理
- 保存用户自定义的提示词，下次启动自动恢复
- 支持命名并保存多套提示词方案（如"工作记录版"、"情感日记版"）

## 影响范围
- `domain/usecase/ExternalAPIProcessor.kt` — 提示词构造逻辑改为动态注入
- `ui/screens/SettingsScreen.kt` — 新增润色档位选择、自定义提示词编辑器
- `data/local/entity or EncryptedPrefs` — 存储用户自定义提示词和档位偏好
- `domain/model/` — 可能需要新增 `GenerationStyle` / `PromptTemplate` 等数据类

## 验收标准
- [x] 提供至少 4 档润色程度，切换后生成效果有明显差异
- [x] 高级用户可编辑完整提示词并保存
- [x] 默认档位（自然润色）比当前版本更克制，不过度改变原意
- [x] 档位和自定义提示词持久化存储，重启后保留
- [x] 单元测试通过（`ExternalAPIProcessorTest` 覆盖 generateDiary 路径）
