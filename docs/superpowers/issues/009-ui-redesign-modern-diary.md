# Issue 009: UI 全面改版 — 参考精致日记应用设计

## 参考设计

参考图片展示了三个核心页面，整体风格特征：
- **配色**：米白/浅灰背景，柔和低饱和色调，大量留白
- **排版**：大字号日期标题，卡片式内容组织，圆角 12-16dp
- **导航**：底部 Tab 导航栏（首页、日历、统计、设置）
- **字体**：细体标题 + 正文字体层级分明

## 当前问题

当前界面过于朴素，缺少日记应用应有的精致感：
- 首页仅展示碎片数量和生成按钮，未展示已有日记内容
- 无底部导航，所有页面靠顶部返回按钮跳转
- 无日历视图，无法直观查看历史记录分布
- 无统计页，PERMA 数据未可视化
- 卡片无阴影/层次，视觉扁平

## 改版方案

### 1. 底部导航栏（新增）
```
┌─────────────────────────────────┐
│  🏠 首页   📅 日历   📊 统计  ⚙️ 设置  │
└─────────────────────────────────┘
```
- 首页：展示今日日记 + 历史日记时间线
- 日历：月份网格，有日记的日期标记圆点/情绪色
- 统计：记录天数、总字数、PERMA 五维雷达图/柱状图、心情分布
- 设置：现有配置页面

### 2. 首页重构
参考左屏设计：
- 顶部大日期标题（"五月 10" / "2024年5月10日 星期五"）
- 今日日记卡片（若已生成）：标题 + 摘要前 3 行 + 字数
- 若无今日日记：展示碎片列表 + "生成今日日记"按钮
- 下方历史日记时间线（按日期倒序），每条显示日期、标题、摘要、字数
- 快速记录 FAB（右下角浮动按钮）

### 3. 日记详情页优化
参考中屏设计：
- 顶部取消大标题 AppBar，改用沉浸式标题（日记标题居中，日期副标题）
- 正文区域加大字号、行高，阅读体验更好
- PERMA 评分改为可视化（五维进度条或小型雷达图）
- 底部操作栏：编辑、分享、复制、删除

### 4. 日历页（新增）
参考右屏上半部分：
- 月份切换头（< 2024年5月 >）
- 7x6 日期网格
- 有日记的日期下方显示小色点（根据当天平均 PERMA 情绪值着色）
- 点击日期跳转该日日记详情

### 5. 统计页（新增）
参考右屏下半部分：
- 总记录天数、总字数大数字展示
- PERMA 五维平均分柱状图/雷达图
- 最近 7 天情绪曲线
- 生成频率统计

### 6. 主题配色调整
- 背景色：#FAF9F6（米白）
- 主色调：#2C3E50（深蓝灰）
- 强调色：#E8A87C（暖橙，用于情绪高值）
- 卡片背景：#FFFFFF 带微弱阴影
- 分割线：极浅灰，弱化存在感

## 依赖考虑

图表库可选方案：
- [YCharts](https://github.com/codeandtheory/YCharts) — 纯 Compose，轻量
- 自绘 Canvas（柱状图/折线图不复杂，可避免引入大库）

- [x] 日历页可查看月份，有日记的日期有标记
- [x] 统计页展示记录天数、字数、PERMA 可视化
